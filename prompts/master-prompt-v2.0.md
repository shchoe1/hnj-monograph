**5.3 DOI 4단계 검증 프로세스 · 각 인용마다 필수**

아래 4단계를 모든 논문 인용에 예외 없이 적용한다. 각 인용에는 최종
\[DOI-VERIFIED ✓\] 태그를 부착한다.

  ---------- ---------------------------- ----------------------------------------------------------------- --------------------------------------------
  **단계**   **작업**                     **확인 방법**                                                     **실패 시 처리**
  1          DOI 존재 확인                doi.org/\[DOI\] 접속 → 200 응답 확인                              인용 배제
  2          제목 일치 검증               ★ DOI 접속 결과 페이지 논문 제목 = 인용 제목 완전 일치 확인       ★ 불일치 시 즉시 배제 (hallucination 위험)
  3          저자·저널·연도 교차 확인     PubMed PMID 조회 · CrossRef API 조회 · 저자·저널·연도 동일 여부   1개라도 불일치 시 배제
  4          원문 접근 · 인용 문장 확인   PDF 또는 Abstract 접근 · 인용한 내용이 실제 원문에 존재 확인      인용 내용 불일치 시 재작성 또는 배제
  ---------- ---------------------------- ----------------------------------------------------------------- --------------------------------------------

**5.3.1 각 인용에 필수 표기할 데이터 필드**

| **인용 표준 형식 (각 인용마다 아래 8개 필드 필수)**              |
|                                                                  |
| \[1\] 저자 (First Author et al.)                                 |
|                                                                  |
| \[2\] 논문 제목 (원문 그대로)                                    |
|                                                                  |
| \[3\] 저널명 (약어 아닌 정식)                                    |
|                                                                  |
| \[4\] 볼륨·페이지·연도                                           |
|                                                                  |
| \[5\] DOI (10.xxxx/xxxxx)                                        |
|                                                                  |
| \[6\] PMID (해당 시)                                             |
|                                                                  |
| \[7\] URL (doi.org/\... 또는 pubmed.ncbi.nlm.nih.gov/\...)       |
|                                                                  |
| \[8\] ★ \[DOI-VERIFIED YYYY-MM-DD ✓\] 검증 완료 태그 · 검증 일자 |

✓\]

**5.4 PubMed PMID 교차 검증**

-   **모든 의학·생물학·영양학 논문은 PubMed PMID 등재 필수**

-   PubMed 미등재 논문: (1) 최근 3개월 이내 · Preprint (bioRxiv·medRxiv)
    표기 허용 or (2) 한국·아시아 지역 저널 · KCI 등재 확인 필요

-   PMID 조회 URL: https://pubmed.ncbi.nlm.nih.gov/\[PMID\]/

-   Cochrane Library ID · PROSPERO 등록번호 등 부가 식별자 함께 표기

**5.5 조선 사료 원전 검증**

-   **이순신 관련 인용 · 특별 검증 필수**

-   난중일기 인용 시: (1) 정확한 연월일 · (2) 원문 (한자·한글 국역)
    · (3) 국역본 페이지 · (4) 국립현충사 or 국사편찬위원회 확인

-   허구·소설·드라마 (예: \'불멸의 이순신\' KBS) 인용 절대 금지

-   최근 학술 논문 · \'이순신의 건강과 질환\' 등 peer-review 논문 우선
    참조

-   **동의보감 · 향약집성방 인용 시**

-   원문 (한자) + 국역 + 편명 (예: 内景篇·脾臟門) + 페이지 (국역본 기준)

-   한국한의학연구원 KTKP DB URL 병기: http://www.oasis.kiom.re.kr/

**5.7 최종 원고 제출 전 자동 검증 스크립트**

-   **각 챕터 원고 완성 시 자동 실행**

-   Python 스크립트 · CrossRef API · PubMed E-utilities API 활용

-   각 인용의 DOI → 논문 제목 실측 → 인용 제목과 문자열 유사도 95%+ 확인

-   검증 실패 인용은 원고에 \[VERIFY-FAILED\] 태그 자동 삽입 · 저자 검토
    필수

-   검증 성공 인용은 \[DOI-VERIFIED YYYY-MM-DD ✓\] 태그 자동 부착

-   **검증 스크립트 의사코드**

  ---------- ---------------------------------------------------------------------------
  **단계**   **코드**
  1          for each citation in chapter:
  2          doi = extract\_doi(citation)
  3          paper = crossref\_api.get(doi)
  4          if paper.title != citation.title (95% similarity): mark \[VERIFY-FAILED\]
  5          if paper.author != citation.author: mark \[VERIFY-FAILED\]
  6          pmid\_paper = pubmed\_api.get(paper.pmid)
  7          if pmid\_paper.doi != doi: mark \[VERIFY-FAILED\]
  8          if all checks pass: mark \[DOI-VERIFIED YYYY-MM-DD ✓\]
  ---------- ---------------------------------------------------------------------------

**7.3 4개 핵심 Agent · Skill 설계**

**7.3.1 Skill 1 · chapter-drafter (챕터 드래프팅)**

-   **역할: 하나의 챕터 (15 섹션) 초고 생성**

-   입력: 챕터 번호 · 주제 원료명 · 마스터 프롬프트 참조

-   출력: chapters/chNN-topic.md (Markdown 원고 · 40-50p 상당)

-   호출 방식: /skill chapter-drafter \--chapter 10 \--topic
    \'L-Theanine\'

-   내부: Task Agent 병렬 실행 (섹션별 서브태스크)

**7.3.2 Skill 2 · reference-validator (참고문헌 검증)**

-   **역할: 원고 내 모든 인용을 DOI 4단계 검증**

-   입력: chapters/chNN-topic.md

-   출력: 각 인용에 \[DOI-VERIFIED ✓\] 태그 부착 · 실패 목록

-   호출 방식: /skill reference-validator \--file ch10.md

-   내부: verify\_doi.py · CrossRef · PubMed API 호출

**7.3.3 Skill 3 · historical-verifier (조선 사료 검증)**

-   **역할: 조선 사료 인용의 원전 정확성 확인**

-   입력: 챕터 내 사료 인용 목록

-   출력: 원전 URL · 페이지 · 원문 확인 태그

-   호출 방식: /skill historical-verifier \--file ch02.md

-   내부: 국사편찬위원회 sillok.history.go.kr · KTKP · 규장각 DB
    크로스체크

**7.3.4 Skill 4 · chapter-editor (편집·통합)**

-   **역할: 챕터 원고의 톤·용어·서술 일관성 확인**

-   입력: 완성된 챕터 원고

-   출력: 편집된 최종 원고 · 편집 로그

-   호출 방식: /skill chapter-editor \--file ch10.md

-   내부: KFDA 표시광고 준수 · 학술 용어 · 참고문헌 스타일 검증

**7.4 챕터별 반복 워크플로우 (5단계)**

  ---------------------- --------------------------------------------- -------------------------------------------- ---------------
  **단계**               **실행자**                                    **산출물**                                   **예상 시간**
  1 · 사전 리서치        리서치 에이전트 (Deep Research)               챕터별 문헌 리스트 100-150편 · 사료 5-20편   1-2주
  2 · 초고 드래프팅      ★ chapter-drafter Skill                       chapters/chNN.md · 40-50p 초고               3-5일
  3 · 참고문헌 검증      ★ reference-validator Skill                   \[DOI-VERIFIED\] 태그 부착 · 실패 목록       1-2일
  4 · 사료 검증          ★ historical-verifier Skill (Ch 1-4 특히)     원전 URL · 페이지 확인                       1-2일
  5 · 저자 검토·에디팅   ★ 저자 (최성화 교수) + chapter-editor Skill   최종 원고                                    1-2주
  ---------------------- --------------------------------------------- -------------------------------------------- ---------------

-   **15 챕터 · 병렬 3-4 챕터 동시 진행 시: 12-15개월**
